CASOS DE PRUEBA EN LA PLATAFORMA TIF STORE

INTRODUCCIÓN
En este documento se puede visualizar detalladamente cada uno de los casos de pruebas que se realizaron en los diferentes procesos
que contiene la plataforma TIF STORE con el fin de encontrar la mayor cantidad de bugs y/o errores para así, remitirlos nuevamente a 
desarrollo y que estos sean corregidos para nosotros como empresa pionera en consultoría IT entregar al cliente un producto de alta calidad.

OBETIVOS
• Realizar diferentes tipos de pruebas funcionales como pruebas de humo, pruebas unitarias, pruebas de integración y pruebas de regresión que permitan identificar bugs en tiempo de desarrollo para así corregirlos durante el desarrollo del producto y tener una mejor gestión de tiempo.
• Aplicar pruebas de regresión automatizadas con Selenium Webdriver ya que estas ocupan menos tiempo que realizarlas de forma manual.
• Verificar el correcto funcionamiento de las Apis haciendo uso del software Postman
• Realizar pruebas de Performance como pruebas de rendimiento, pruebas de carga, pruebas de seguridad y pruebas de estrés.

HERRAMIENTAS Y/O TECNOLOGÍAS UTILIZADAS PARA LOS CASOS DE PRUEBA
• Selenium WebDriver
• ChromeDriver v110.0.5481.77
• Postman
• Jmeter
